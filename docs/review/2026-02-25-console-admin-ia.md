# 控制台 / 后台信息架构收敛（Issue #50）

## 方向修正（按产品意见）

> dashboard 就应该做 dashboard 的工作。

本次已按该原则调整：

1. 职责分离
- `/dashboard`：仅保留内容看板与文章工作流（统计、筛选、发布、站点配置入口）
- `/admin`：承载后台管理能力（用户体系、个人介绍配置、用户管理、分类管理、审计日志）

2. 入口与导航
- 顶部导航管理员入口恢复为“后台管理”并指向 `/admin`
- 不再把后台模块塞入 dashboard tab

3. 代码结构
- 后台能力仍复用 `components/admin-workspace.tsx`，避免逻辑重复
- 页面分工回归清晰：Dashboard ≠ Admin

## 结果
- 解决“控制台和后台混在一起”的认知负担
- 维持后台代码模块化与可维护性

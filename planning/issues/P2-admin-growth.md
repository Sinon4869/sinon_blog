# [P2] 管理后台与增长体系（第 3 周）

## 目标
补齐运营后台与增长闭环，形成长期可维护系统。

## 范围
- Admin 用户/内容管理
- 订阅通知与数据分析
- 安全治理（限流、合并流程、轮换）

## 任务拆解
- [ ] 用户管理：禁用/解禁、角色调整、最近登录
- [ ] 内容管理：草稿、定时发布、回收站
- [ ] 管理员操作审计日志
- [ ] 邮件或机器人订阅推送
- [ ] 隐私友好统计（页面、来源、搜索词）
- [ ] OAuth 账号安全合并（密码二次确认）
- [ ] 登录/注册/评论限流
- [ ] secrets 轮换流程文档化（Google/Cloudflare）

## 验收标准
- [ ] 管理后台具备基本运营能力
- [ ] 发布通知链路稳定
- [ ] 高风险操作具备审计与追踪
